# 网页设计课程期末大作业 — 项目报告

---

## 项目标题

**网易云音乐客户端仿站** —— 使用原生 HTML / CSS / JavaScript 实现的网易云音乐桌面端高仿界面

---

## 内容简介

本项目完整复刻了网易云音乐桌面客户端的主要界面与交互功能，共包含 **7 个功能页面**：推荐首页、精选、播客、私人漫游、关注动态、我喜欢的音乐、最近播放。

项目内置 5 首带真实音源的歌曲，并实现了一个**跨页面续播**的共享播放引擎：歌曲、播放进度、音量、播放模式、喜欢状态等全部通过 `localStorage` 持久化，在任意页面间跳转时都能无缝继续播放，最大限度还原了真实客户端的使用体验。

页面采用**左侧侧边栏 + 顶部导航栏 + 中间内容区 + 底部播放控制栏**的经典四栏布局，视觉风格严格对齐网易云音乐官方客户端。

---

## 项目特色

1. **真实可播放**：内置 5 首带真实音源的歌曲，支持播放/暂停、上一首/下一首、进度拖拽、音量调节、静音等完整播放功能。
2. **跨页面续播**：单例 `Audio` + `localStorage` 方案，切换页面不中断播放，自动恢复进度与状态。
3. **网易云风格进度条**：底栏顶部红色进度条，悬停加宽、白色拖拽圆点，并在红条末端显示当前播放时间气泡。
4. **三种播放模式**：顺序播放 / 随机播放 / 单曲循环，一键切换。
5. **喜欢（红心）状态**：按歌曲记忆，跨页面同步。
6. **响应式细节**：卡片网格随窗口宽度自动调整列数，统一的滚动条、搜索框聚焦、按钮点击反馈等 UI 细节。
7. **丰富内容页**：推荐、精选、播客、漫游、关注动态、我喜欢的音乐、最近播放等多个功能页面，内容充实。
8. **共享播放引擎**：所有页面共用 `js/player.js`，页面只需放置约定好的元素 ID，播放引擎自动绑定底栏控件并注入顶部进度条，无需重复编写播放逻辑。

## 首　页（index.html）

**1、文件名：** `index.html`

**2、作用：** 推荐首页，是用户进入网站后的第一个页面。展示轮播横幅、推荐歌单网格、精选活动等内容，同时作为整个应用的主框架页面，包含左侧侧边栏导航、顶部搜索栏和底部播放控制栏。

**3、主要技术：**

- CSS 变量（`:root`）统一管理全局主题色、间距、布局尺寸
- CSS Grid 实现推荐歌单卡片网格布局，响应式自适应列数
- CSS Flexbox 实现四栏布局（侧边栏 + 顶部栏 + 内容区 + 底部播放栏）
- JavaScript 动态渲染推荐歌单卡片（数据驱动 DOM 生成）
- JavaScript 实现轮播横幅自动播放与手动切换
- 共享播放引擎 `player.js` 绑定底栏控件

**4、素材：**

- 推荐歌单封面图：`images/rec*.jpg`
- 歌手头像：`images/singers/`
- 歌曲封面：`images/songs/`
- 网易云 Logo：`images/logo.png`
- 歌曲音源：`audio/` 目 录下 5 首 mp3/m4a 文件

## 项目难点及解决方法

### 难点一：跨页面续播的实现

**问题描述：** 浏览器中每个 HTML 页面拥有独立的执行环境，页面切换时原有的 `Audio` 对象和 JavaScript 状态会丢失，导致音乐中断。

**解决方法：** 设计了**单例 Audio + localStorage 持久化**方案。`player.js` 在页面加载时从 `localStorage` 读取上次播放的歌曲索引、进度、音量、播放模式、喜欢状态等信息，自动恢复到上次的播放状态。所有状态变更都实时写入 `localStorage`，确保任意页面跳转后都能无缝续播。

### 难点二：共享播放引擎的设计

**问题描述：** 7 个页面都需要底部播放控制栏，如果每个页面各自实现播放逻辑，代码会大量重复且难以维护。

**解决方法：** 将播放逻辑抽取为独立的 `js/player.js` 模块，暴露全局 `window.NeteasePlayer` 接口。各页面只需在底栏放置约定好的元素 ID（如 `trackName`、`playBtn`、`volumeContainer` 等），播放引擎会自动绑定控件并注入顶部进度条，页面脚本只需调用 `NeteasePlayer.playByIndex(i)` 等方法即可驱动播放。

### 难点三：网易云风格 UI 的高保真还原

**问题描述：** 网易云音乐客户端的 UI 细节丰富（红色进度条、悬停加宽、时间气泡、卡片阴影等），用纯 CSS 实现有一定难度。

**解决方法：** 通过 Chrome DevTools 逐像素对照官方客户端，使用 CSS 伪元素（`::before`、`::after`）、`transition`、`box-shadow`、`border-radius` 等属性精细还原。公共样式提取到 `css/common.css` 中统一管理，保证跨页面视觉一致。

### 难点四：响应式卡片网格布局

**问题描述：** 歌单卡片需要根据窗口宽度自动调整列数，同时保持卡片的宽高比和间距一致。

**解决方法：** 使用 CSS Grid 的 `repeat(auto-fill, minmax(150px, 1fr))` 实现自适应列数，配合 `aspect-ratio: 1` 保持封面正方形比例。JavaScript 监听 `resize` 事件，在窗口大小变化时重新计算布局。

## 收获与思考

### 技术收获

1. **原生 JavaScript 功底提升**：通过从零实现播放引擎、数据持久化、DOM 动态渲染等功能，深入理解了 `Audio` API、`localStorage`、事件委托、发布-订阅模式等核心概念。
2. **CSS 布局能力增强**：熟练掌握了 CSS Grid 和 Flexbox 两种现代布局方式，能够灵活应对复杂的多栏布局需求。
3. **模块化思维**：将播放逻辑抽取为独立模块（`player.js`），实现了代码复用和关注点分离，体会到了模块化设计的好处。
4. **UI 还原能力**：学会了使用开发者工具逐像素对照、分析盒模型、调试样式，提升了从设计稿到代码的转化能力。

### 团队协作与项目管理

- 学会了在项目初期做好需求分析和页面规划，避免后期频繁返工
- 素材收集与整理同样重要，合理的目录结构能提高开发效率

### 不足与改进方向

1. 本项目未使用任何前端框架（如 Vue、React），在大型项目中组件化和状态管理会成为瓶颈，后续可尝试用框架重构
2. 当前歌曲数据硬编码在 JS 中，实际项目应通过后端 API 动态获取
3. 部分页面的内容为静态模拟数据，未接入真实后端接口
4. 可进一步优化移动端适配，目前主要面向桌面端

### 总结

本次期末大作业让我将课堂所学的 HTML、CSS、JavaScript 知识综合运用到了一个完整的项目中，从需求分析、页面设计、功能开发到最终测试，经历了一个较为完整的 Web 开发流程。最大的收获是理解了"**前端开发不仅仅是写页面，更是对用户体验和交互细节的极致追求**"。
