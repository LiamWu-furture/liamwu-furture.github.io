# 网易云音乐仿站

> 网页设计课程期末大作业 —— 使用原生 HTML / CSS / JavaScript 实现的网易云音乐客户端高仿界面。

本项目复刻了网易云音乐桌面客户端的主要界面与交互，包含 7 个功能页面，并实现了一个**跨页面续播**的共享播放引擎：歌曲、播放进度、音量、播放模式、喜欢状态等全部通过 `localStorage` 持久化，在任意页面间跳转时都能无缝继续播放。

---

## 功能特性

- **真实可播放**：内置 5 首带真实音源的歌曲，支持播放/暂停、上一首/下一首、进度拖拽、音量调节、静音。
- **跨页面续播**：单例 `Audio` + `localStorage`，切换页面不中断播放，自动恢复进度与状态。
- **网易云风格进度条**：底栏顶部红色进度条，悬停加宽、白色拖拽圆点，并在红条末端显示当前播放时间气泡。
- **三种播放模式**：顺序播放 / 随机播放 / 单曲循环。
- **喜欢（红心）状态**：按歌曲记忆，跨页面同步。
- **响应式细节**：卡片网格随窗口宽度自动调整列数，统一的滚动条、搜索框聚焦、按钮点击反馈等 UI 细节。
- **丰富内容页**：推荐、精选、播客、漫游、关注动态、我喜欢的音乐、最近播放等。

## 运行方式

本项目为纯静态页面，无需构建或安装依赖。

### 方式一：直接打开
双击任意 `.html` 文件用浏览器打开即可。

> 注意：以 `file://` 方式打开时，部分浏览器对 `localStorage` 有限制（已在代码中做了容错），建议使用本地服务器获得最佳体验。

### 方式二：本地服务器（推荐）

```bash
# Python 3
python -m http.server 8000

# 或 Node.js
npx serve
```

然后在浏览器访问 `http://localhost:8000/index.html`。

## 技术说明

### 共享播放引擎 `js/player.js`

所有页面引入同一个 `player.js`，它在 `window.NeteasePlayer` 上暴露统一接口：

| 接口 | 说明 |
|------|------|
| `playlist` | 核心歌单（唯一数据源） |
| `playByIndex(i)` | 播放指定索引歌曲 |
| `toggle()` | 播放 / 暂停 |
| `next()` / `prev()` | 下一首 / 上一首 |
| `seek(ratio)` | 按比例跳转进度 |
| `setVolume(v)` / `toggleMute()` | 音量 / 静音 |
| `cycleMode()` | 切换播放模式 |
| `toggleLike()` / `isLiked()` | 喜欢状态 |
| `indexOfName(name)` | 按歌名匹配核心歌单索引 |
| `onChange(cb)` / `onProgress(cb)` | 订阅状态 / 进度变化 |
| `getState()` | 获取当前状态快照 |

页面只需在底栏放置约定好的元素 ID（如 `trackName`、`playBtn`、`volumeContainer` 等），`player.js` 会自动绑定底栏控件并注入顶部进度条，无需重复编写播放逻辑。

### 样式组织

- 每个页面有各自的内联 `<style>`，负责该页独有的布局。
- `css/common.css` 抽取全站统一的 UI 细节（滚动条、播放器底栏、音量条、卡片交互、响应式网格、搜索框聚焦等），保证跨页面视觉一致。
